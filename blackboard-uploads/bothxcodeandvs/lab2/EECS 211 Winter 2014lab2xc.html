Machine Problem MP#1: Partition a Command Line

Due: 1/29/2014 at 11:59pm

Objective:

The purpose of this MP is to write a tool C program which can partition a command line into segments. You will gain some experience in:

[1] basic stream I/O;
[2] C/C++ control structures;
[3] processing char arrays;
[4] VC++ programming environment.
 

Problem Description

Write a program that will read and parse a command line from the keyboard.  More specifically, your program should do the following:

  • Print out a welcome message once.
  • Prompt the user to enter a command line.
  • Partition the command line into segments.
  • Print each command segment on a different line.
  • Iterate the partitioner until you input "quit"
    • A valid segment is one of the following cases:
    1. a name of a variable, e.g., abc, var_1, ptr_1
    2. an operator, =, +, -, *, /, ++, --
    3. a number, e.g., 1, 1.3, 3.1415926
    4. an array of numbers in [ ],  e.g., [2 3 4], [1 3 4; 2 3 5; 2 4 5]

    To be more specific:

    1. A variable is any sequence of letters, digits and underscore. It starts with a letter.
    2. Only consider the following operators : =, +, -, *, /, ++, --.
    3. Only integers and decimal numbers are acceptable as numbers.
    4. An array of numbers in [ ] can be treated as matrix. Blanks or commas are used to separate the numbers in a row. And semicolons are used to separate rows. You have to verify that the elements within [ ] are numbers or valid separators. In this project, you don't have to check if it is a valid matrix. But in a later project, you will do.
    5. The number and position of blanks in the line can be arbitrary.
    If a line contains a segment that is not any of the above cases, detect it and print out an error message, e.g., "Sorry, don't understand!".
