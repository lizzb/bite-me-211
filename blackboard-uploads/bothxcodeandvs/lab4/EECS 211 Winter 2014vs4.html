Objective:

In Lab 4, we will build classes for use later in the project. In this assignment, you will learn to:

[1] Work with multiple source files
[2] Practice dynamic memory allocation
[3] Design a class, including its data members and member functions
[4] Implement class member functions, including constructors and destructors
[5] Test and debug classes
 

Problem Description

We will build a CMatrix class to perform matrix operations. Your CMatrix should implement the following functions:

  • CMatrix(): constructs a "null" (invalid) matrix
  • CMatrix(double d): constructs a 1 x 1 CMatrix with the given value
  • CMatrix(int nrows, int ncol): constructs a CMatrix with nrows rows and ncols columns with all zero entries; the matrix is set to a null state if either of these are less than or equal to zero
  • CMatrix(const char* str): constructs a CMatrix given a string input (or a null matrix if the string is invalid)
  • bool isNull(): returns whether or not the CMatrix is invalid
  • void swap(CMatrix& other): swaps the contents of this CMatrix with the given CMatrix
  • double& element(int row, int col): returns the value in the indicated row and column
  • int getNRows(): returns the number of rows in the CMatrix
  • int getNCols(): returns the number of columns in the CMatrix
  • void resize(int newRows, int newCols): resizes the CMatrix to the given size, keeping all existing elements and filling additional elements with zeros
  • ~CMatrix(): destroys the CMatrix object, deallocating any allocated memory

In the same file, you should also implement a void printMatrix(CMatrix& m) function that prints the given CMatrix to cout in the format described in the attached output.jpg.

You should use assert() statements to test for errors in the non-constructor member functions. Constructors should initialize the CMatrix to a "null" state in the event of an error.

You should submit two files: the cmatrix.cpp source file, which should implement the above functions; and a cmatrix.h header file that defines the CMatrix class. We have provided a sample cmatrix.h file with this assignment; you should add the line
#include "cmatrix.h"
near the top of your cmatrix.cpp file. You may change the data members or add private/protected member functions to the provided cmatrix.h, but you should not change the definitions of the required member functions.

When implementing your class, you may use either a double** (array of arrays) or a double* array to represent the values in the CMatrix. When using a double* to represent a 2D array, you can access element (i, j) of the matrix at index [i * nCol + j] in the double* array.

Also, it is easier to locate bugs if you test your code piece by piece instead of implementing all of the functions at the same time. For this reason, you may wish to write your own matrix_driver.cpp file that constructs various CMatrix objects and tests the functions you implement, adding more and more tests as you implement more constructors and functions for the CMatrix class. (This file should also include cmatrix.h.) You may wish to start implementing your class with the default constructor and print function and implement more complex functions like the string constructor and resize function last.

Some example test cases are provided in example_test.cpp.
